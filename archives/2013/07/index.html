Welcome to SparkleMan Blog!

This blog builds based on Hexo! Check documentation to learn how to use.

准备阶段

Hexo是基于NodeJS实现，用于快速撰写和发布blog，支持Markdown，支持发布到GitHub Pages。开发作者是来自台湾。 环境搭建及使用可以参见Hexo官方文档，已经写得非常清楚了。

撰写Blog

可以通过如下命令创建新的文章

hexo new "New Post" -> source/_posts/new-post.md

hexo new page "New Page" -> source/new-page/index.md

hexo new draft "New Draft" -> source/_drafts/new-draft.md

撰写Blog时候，可以启用hexo自带server，就可以本地查看自己写的blog，如果有修改，要自己手动再刷新下浏览器，不支持blog文章变化了自动触发浏览器刷新。

hexo server [-p port] 默认端口4000: http://localhost:4000

部署到GitHub

这一过程碰到的最大问题，就是部署完之后，blog页面导航路径不对以及一些静态资源引用路径不对，这里需要看下自己theme的配置及其它路径配置是否正确。 此外，除了基于hexo提供的deploy命令来部署到GitHub，其实也可以自己拉自己的仓库到本地，然后本地初始化自己的blog之后通过git提交。

产生静态资源文件：hexo generate

部署到GitHub：hexo deploy

部署前重新产生一次静态资源文件：hexo deploy --generate

此外，还要注意的一点，通过命令行部署时候，有时候会提交失败，把当前.deploy目录删除，重新来一遍，就可以正常提交了。
